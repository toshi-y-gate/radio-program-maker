# ラジオ番組メーカー — 要件定義書

**プロジェクト名**: ラジオ番組メーカー（AI Voice Radio Creator）
**作成日**: 2026-03-21
**バージョン**: 1.0
**ステータス**: Phase 1 完了 / Phase 2 以降 計画中

---

## 1. プロジェクト概要

### 1.1 目的
トークスクリプトを入力するだけで、AIボイス（音声クローン含む）を使った高品質なラジオ番組を自動生成するWebアプリケーション。

### 1.2 ターゲットユーザー
- ラジオ番組制作者
- Podcast配信者
- 企業の社内放送・研修コンテンツ制作者
- 英会話スクール等の教育コンテンツ制作者
- 非エンジニア（直感的なWeb UIで操作可能）

### 1.3 ビジネス目標
- 将来的に商用サービスとして展開
- 想定ユーザー規模: 最大10,000名
- サブスクリプション型の課金モデル

## 2. システム構成

### 2.1 技術スタック

| 項目 | 技術 |
|------|------|
| 言語 | Python 3.12 |
| Web UI | Streamlit |
| 音声合成API | MiniMax TTS API (Speech 2.8) |
| 音声処理 | pydub |
| 設定管理 | python-dotenv |
| 将来のバックエンド | FastAPI（商用化フェーズ） |

### 2.2 アーキテクチャ

```
[ユーザー] → [Web UI (Streamlit)]
                    │
                    ├── スクリプト解析 (script_parser.py)
                    ├── キャッシュ確認 (cache.py)
                    │       ├── HIT → キャッシュから返却（API呼び出しなし）
                    │       └── MISS → MiniMax TTS API 呼び出し (tts.py)
                    ├── 音声結合・BGM合成 (audio_mixer.py)
                    └── MP3/WAV 出力
```

### 2.3 ディレクトリ構成

```
音声クローンアプリ/
├── app.py                  # Web UI（Streamlit）
├── core/
│   ├── config.py           # 設定・ボイスプリセット
│   ├── tts.py              # MiniMax API連携
│   ├── script_parser.py    # スクリプト解析
│   ├── audio_mixer.py      # BGM合成・音声結合
│   └── cache.py            # キャッシュ管理
├── bgm/                    # BGMファイル置き場
├── cache/                  # 音声キャッシュ
├── output/                 # 生成音声の出力先
├── docs/                   # ドキュメント
├── .env                    # APIキー（Git管理外）
├── .gitignore
├── requirements.txt
└── venv/                   # Python仮想環境
```

## 3. 機能要件

### 3.1 Phase 1（✅ 完了）

#### F1-1: スクリプト入力・解析
- テキストエリアにスクリプトを入力
- 対応フォーマット:
  - `[話者名] セリフ`
  - `【話者名】 セリフ`
  - `話者名: セリフ`
  - `話者名： セリフ`
- 話者の自動検出（最大4名）
- スクリプトプレビュー表示

#### F1-2: 音声合成（MiniMax API連携）
- MiniMax TTS API (Speech 2.8 HD/Turbo) による音声生成
- プリセットボイス選択（日本語4種、英語4種）
- 音声パラメータ調整:
  - 話速（0.5〜2.0倍）
  - 音量（0.1〜10.0）
  - ピッチ（-12〜+12）
  - 感情（calm, happy, sad, angry, surprised, fluent, whisper）
- API接続テスト機能

#### F1-3: 話者管理
- 各話者へのボイス割り当て
- 最大4名の同時話者対応
- 話者ごとに異なるボイスを設定可能

#### F1-4: 音声後処理
- 複数セリフの自動結合（セリフ間にポーズ挿入）
- 音量正規化
- BGM合成:
  - BGMファイルアップロード（MP3/WAV/OGG）
  - 挿入モード: 番組全体 / 番組前のみ / 番組後のみ / 番組前後
  - BGM音量調整（-30〜0 dB）
  - イントロ・アウトロ長さ調整
  - 自動ループ・フェードイン/アウト

#### F1-5: 出力
- ブラウザ上で再生プレビュー
- MP3ダウンロード
- 音声長さ・話者数の表示

#### F1-6: キャッシュ機能
- テキスト+ボイス+設定の組み合わせをハッシュ化
- 同一条件の再生成時はAPIを呼ばずキャッシュから返却
- キャッシュ統計表示（件数・容量）
- キャッシュクリア機能

#### F1-7: Web UI
- Streamlitベースのレスポンシブ画面
- サイドバー: 設定・音声パラメータ・BGM・キャッシュ
- メインエリア: 番組作成タブ / 使い方タブ
- 進捗バー付き生成プロセス

## 4. 非機能要件

### 4.1 パフォーマンス
- 1セリフの音声生成: 5秒以内（API応答時間依存）
- 10分番組の全体生成: 2分以内
- キャッシュヒット時: 即座に返却

### 4.2 セキュリティ
- APIキーは `.env` で管理（Git管理外）
- APIキーのハードコーディング禁止
- 将来: ユーザー認証、通信のHTTPS化

### 4.3 コスト管理
- キャッシュによるAPI呼び出し削減（目標: 30〜60%削減）
- Turbo/HDモデルの使い分けによるコスト最適化
- 将来: MiniMax Custom契約による単価交渉

### 4.4 スケーラビリティ
- Phase 1: ローカル環境（単一ユーザー）
- Phase 3: マルチユーザー対応（〜1,000名）
- Phase 4: 商用スケール（〜10,000名）

## 5. 外部サービス依存

### 5.1 MiniMax API

| 項目 | 詳細 |
|------|------|
| エンドポイント | `https://api.minimax.io/v1/t2a_v2` |
| 認証 | Bearer Token（APIキー） |
| モデル | speech-2.8-hd / speech-2.8-turbo |
| テキスト上限 | 10,000文字/リクエスト |
| 対応形式 | MP3, WAV, FLAC, PCM |
| 課金 | 文字数ベース従量課金 |

### 5.2 コスト見積もり

| 規模 | 月間文字数 | 月間コスト（HD） | キャッシュ後 |
|------|-----------|------------------|-------------|
| テスト（1人） | 〜50,000 | 〜$5（750円） | 〜$2 |
| 小規模（100人） | 〜5,000,000 | 〜$500（75,000円） | 〜$200 |
| 中規模（1,000人） | 〜50,000,000 | 〜$5,000（75万円） | 〜$2,000 |
| 大規模（10,000人） | 〜500,000,000 | 〜$50,000（750万円） | 〜$20,000 |

※大規模時はCustom契約で大幅割引交渉、OSSモデル併用で更に削減可能

## 8. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| MiniMax API障害 | サービス停止 | フォールバック用のサブAPIを検討 |
| API料金値上げ | コスト増加 | キャッシュ強化、OSSモデル併用 |
| 音声品質の不満 | ユーザー離脱 | 複数モデル選択肢、ボイスクローン提供 |
| APIキー漏洩 | 不正利用 | 環境変数管理、キーローテーション |
| スケーリング限界 | 応答遅延 | キュー処理、非同期生成の導入 |
